# Security Policy

## Supported Versions

We provide security updates for the latest released version of Theo OpenVoice.
If you are using an older version, please upgrade before reporting a
vulnerability.

## Reporting a Vulnerability

Please report security issues privately by emailing security@usetheo.com.

Do not open public GitHub issues for security vulnerabilities.

When reporting, include:

- A description of the issue and potential impact
- Steps to reproduce or a proof-of-concept
- Affected version(s)

## Response Timeline

We aim to acknowledge reports within 72 hours and provide a resolution or
mitigation plan as soon as possible.
